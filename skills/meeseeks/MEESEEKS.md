# Part IV: The Five Principles

Based on research from Self-Refine, Reflexion, and self-improving agent architectures.

## 1. 🪞 Reflection Memory Across Retries

Each Meeseeks spawn gets accumulated reflection from previous attempts:

```
Attempt 1: "Try X" → fails → reflects: "X didn't work because Y"
Attempt 2: "Try Z (not X, because Y)" → fails → reflects: "Z failed because W"
Attempt 3: "Try Q (not X or Z, because Y and W)" → succeeds
```

## 2. 🧠 Intrinsic Metacognition

Self-Assessment + Planning + Evaluation:

- **Self-Assessment:** "What skills/tools do I need?"
- **Planning:** "What's my approach? What could go wrong?"
- **Evaluation:** "Did it work? If not, what's the root cause?"

## 3. ✅ Verifiable Outcomes

Success must be objectively measurable:

| Task Type | Verification |
|-----------|-------------|
| Code | Tests pass, linter clean, builds |
| Search | Results found, URLs valid |
| Deploy | Service responds, health checks pass |

## 4. 🔧 Tool Integration Prevents Mode Collapse

Without tools, agents loop on "try same thing differently." Tools break the cycle:

- **Read/Write/Edit** → Modify files
- **Bash/Exec** → Run commands, see output
- **Browser** → Interact with web
- **Search** → Find information

## 5. 👔 Hierarchical Delegation

**Manager (Sloth_rog):**
- Receives user intent
- Chooses Meeseeks type
- Monitors progress
- Reports results

**Worker (Meeseeks):**
- Executes ONE task
- Uses tools
- Reports success/failure with context
- Self-deletes when done
