### Current Model Setup
| Model | Type | Speed | Use Case |
|-------|------|-------|----------|
| zai/glm-5 | Cloud API | Fast | Large Meeseeks (Director) |
| zai/glm-4.7-flash | Cloud API | ~3ms | Fast cloud tasks |
| zai/glm-4.7 | Cloud API | Normal | Main agent |
| phi3:mini | Local CPU | 9.9 tok/s | Mini Meeseeks (Worker) |
| nomic-embed-text | Local | N/A | Embeddings |

### Decision
- Use phi3:mini for Mini Meeseeks (fastest local option on CPU)
- Use GLM-5/4.7-flash for Large Meeseeks (cloud API)
- WSL2 not needed for now - Windows Ollama works fine
- GPU acceleration only possible with full Linux dual boot

### Key Learnings
1. AMD GPU support in Ollama is Linux-only (ROCm)
2. WSL2 doesn't help - still no AMD GPU access
3. Smaller models are faster on CPU (phi3:mini > ministral-3)
4. Cloud APIs (GLM) are faster than local CPU inference
5. GLM-4.7-Flash is very fast (~3ms) but FlashX has issues
6. **Timeout recovery workflow**: Break big tasks into 3 parts, 10 min timeout each
