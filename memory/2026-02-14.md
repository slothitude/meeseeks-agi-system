## OpenWebUI Integration Request (Late Night Session)

User asked: "could we make it operate with openclaw" - referring to https://docs.openwebui.com/

### Research Findings

**OpenWebUI:**
- Self-hosted AI web interface platform
- Supports Ollama and OpenAI-compatible APIs (Chat Completions protocol)
- Can connect to OpenAI, LMStudio, GroqCloud, Mistral, OpenRouter, etc.
- Custom OpenAI API URL configuration for various providers
- Rich features: RAG, web search, image generation, multiple model conversations

**OpenClaw:**
- Gateway-based WebSocket control plane (ws://127.0.0.1:18789)
- Does NOT expose OpenAI-compatible API by default
- Connects to chat apps (WhatsApp, Telegram, Discord, etc.)
- Tools: browser, canvas, nodes, cron, sessions
- Uses internal WebSocket protocol for agent communication

### Integration Approaches Proposed

**Option 1: OpenAI-compatible API Proxy (Recommended)**
- Write a custom OpenClaw skill that exposes HTTP endpoint at `/v1/chat/completions`
- Translates OpenAI-format requests to OpenClaw agent calls
- Streams responses back in OpenAI format
- OpenWebUI sees it as standard OpenAI-compatible provider
- Cleanest separation of concerns

**Option 2: Webhook Integration**
- Use OpenWebUI's webhook support
- Send webhooks from OpenWebUI to OpenClaw
- OpenClaw responds via messaging channels or triggers tasks
- Less direct, more async

**Option 3: Direct Session API**
- Use OpenClaw's `sessions_*` tools from OpenWebUI
- If OpenWebUI can run external Python (it does)
- Write script to spawn sessions and get responses via internal API

### Decision
Offered to prototype a basic OpenAI-compatible proxy skill (Option 1) as the cleanest approach. Pending user decision.
