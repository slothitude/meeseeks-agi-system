### Key Findings

#### Parallel Execution Patterns
1. **Map-Reduce**: Split work → parallel process → aggregate results
2. **Fan-out/Fan-in**: Distribute tasks to workers → collect results
3. **Work Stealing**: Dynamic load balancing, idle workers steal from busy ones
4. **Hierarchical**: Manager assigns work based on worker capabilities

#### Coordination Strategies
- **Shared State Store**: JSON file with progress, results, locks, events
- **Claim-based Assignment**: Atomic task claiming prevents duplicate work
- **Hash-based Partitioning**: Deterministic task assignment

#### Conflict Resolution
- Advisory locks for file access
- Atomic writes (write to temp, rename)
- Scoped output directories per worker

#### Error Handling
- **Retriable errors**: Network, rate limits, locks (exponential backoff)
- **Fatal errors**: Invalid spec, missing deps (fail fast)
- **Circuit breaker**: Prevent cascading failures
