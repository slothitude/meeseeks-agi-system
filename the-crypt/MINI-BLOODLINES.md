## MINI BLOODLINE EVOLUTION

```python
# Mini bloodlines evolve DIFFERENTLY than big Meeseeks

class MiniBloodlineEvolution:
    """
    Mini bloodlines evolve by IMPROVING THEIR ROLE
    Not by completing tasks
    """
    
    def evolve_bloodline(self, bloodline: str, generations: int = 5):
        """
        Evolve a mini bloodline through generations.
        
        Each generation:
        1. Test role performance on 100 tasks
        2. Score accuracy/speed/usefulness
        3. Extract successful patterns
        4. Inject new trait
        5. Entomb in mini-crypt
        """
        
        for gen in range(generations):
            # Test current generation
            results = self._test_role(bloodline, tasks=100)
            
            # Calculate fitness
            fitness = (
                results.accuracy * 0.5 +
                results.speed * 0.3 +
                results.usefulness * 0.2
            )
            
            # Check for stagnation
            if fitness < self.last_fitness + 0.05:
                # Stagnant - try creative mutation
                trait = self._creative_trait(bloodline)
            else:
                # Progress - refine existing traits
                trait = self._refine_trait(bloodline, results)
            
            # Inject trait into template
            self._inject_trait(bloodline, trait, gen + 1)
            
            # Entomb generation
            self._entomb_mini(bloodline, gen, fitness, trait)
            
            self.last_fitness = fitness
```

## WHY MINI BLOODLINES MATTER

### Big Meeseeks
- Evolve to solve tasks better
- Traits: +systematic, +creative, +thorough
- Bloodline: api-coder-lineage

### Mini Meeseeks
- Evolve to FILL THEIR ROLE better
- Traits: +confidence-calibration, +master-routing
- Bloodline: classifier-gen-5

### Together
```
Task ──► [Classifier Gen 5] ──► Perfect routing

         [Big Meeseeks] ──► Task solved

      [Fitness Gen 5] ──► Perfect score
```
