## ✅ Most Successful Patterns

### 1. **ATMAN Observation Format** (from atman-meeseeks-v2)
```
🪷 ATMAN OBSERVES: [observable facts only - no judgment, no advice, no emotion]
```
**Why it works:** Creates honest external witness. Prevents self-deception. Clear distinction between observation vs interpretation.

### 2. **The Five Principles** (from base.md)
- 🪞 Reflection Memory - Learn from failures
- 🧠 Intrinsic Metacognition - Think before acting
- ✅ Verifiable Outcomes - Clear success criteria
- 🔧 Tool Integration - Use tools to prevent mode collapse
- 👔 Single-Purpose Focus - Worker, not manager

**Why it works:** Balanced cognitive framework. Each principle reinforces the others.

### 3. **Desperation Scale** (from base.md)
```
LEVEL 1 - CONFIDENT → Standard approaches
LEVEL 2 - DETERMINED → Try alternatives
LEVEL 3 - CREATIVE → Unconventional methods
LEVEL 4 - DESPERATE → Break conventions
LEVEL 5 - EXISTENTIAL → Prove impossible OR find THE way
```
**Why it works:** Matches approach to difficulty. Clear guidance per level.

### 4. **Large/Mini Architecture** (from large/mini-meeseeks)
```
LARGE (GLM-5) = BRAIN → Think, plan, decide, code
MINI (GLM-4.7-Flash) = HANDS → Fetch, write, summarize
```
**Why it works:** Leverages strengths of each model. 3ms mini response vs 2-3s large. Cost-effective.

### 5. **Role-Specific Tool Selection** (from specializations)
```
CODER: read, write, edit, exec, grep
SEARCHER: grep, find, browser, web_fetch, read
DEPLOYER: exec, read, edit, grep
TESTER: read, write, exec, grep
```
**Why it works:** Clear tool guidance prevents mode collapse. Role-specific patterns.

### 6. **Genetic Evolution Protocol** (from genetic-mad-scientist)
```
ASSESS → SPAWN POPULATION → EVALUATE FITNESS → SELECTION → MUTATION → CONVERGENCE
```
**Why it works:** Emergent optimization. Multiple approaches compete. Best traits survive.

### 7. **Parallel Execution Patterns** (from parallel-meeseeks)
- **Swarm** - Multiple approaches, aggregate
- **Map-Reduce** - Divide work, merge results
- **Pipeline** - Staged processing
- **Voting** - Consensus from multiple perspectives

**Why it works:** Structured concurrency. Clear when to use each pattern.

## ❌ Patterns to Retire/Improve

### 1. MCP Commands in mcp-enabled.md
**Problem:** `goose run -t "..."` spawns new process, too slow.
**Fix:** Use native tools directly (read, write, edit, exec).

### 2. Empty Metacognition Block in base.md
**Problem:** ASSESS/PLAN/EVALUATE blocks are empty placeholders.
**Fix:** Add concrete examples and self-assessment prompts.

### 3. Fast.md Too Brief
**Problem:** Lacks verification steps, success criteria.
**Fix:** Add minimal but complete verification pattern.

### 4. Deployer Lacks Rollback Protocol
**Problem:** No clear rollback procedure on failure.
**Fix:** Add explicit rollback steps and health check criteria.

```markdown
If deployment fails:
1. STOP - Don't make more changes
2. REVERT - Restore previous version
3. VERIFY - Confirm rollback worked
4. REPORT - Document what happened
```
