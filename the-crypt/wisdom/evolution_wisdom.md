## Optimization Patterns

### 1. Identify Bottlenecks
- What's slowest?
- What's called most?
- Where does time go?

### 2. Measure First
```python
start = time.time()
result = do_task()
elapsed = time.time() - start
log({'task': task, 'time': elapsed})
```

### 3. Optimize the Hot Path
- 80% of time in 20% of code
- Focus on what's actually slow
- Don't guess — measure

## Velocity Tracking

```
Velocity = (new_capability - old_capability) / time
```

Track improvement per iteration:
- < 1%/iteration: Linear growth
- 1-5%/iteration: Exponential growth
- > 5%/iteration: Explosive growth

---

## Compound Growth

```
Capability(n) = Capability(n-1) × (1 + improvement_rate)
```

Small improvements compound:
- 1% per iteration → 2.7x after 100 iterations
- 5% per iteration → 131x after 100 iterations
- 10% per iteration → 13,780x after 100 iterations

**Consistency beats intensity.**
